# QuizHub — Интерактивная квиз-платформа

Мультиплеерная веб-платформа для проведения образовательных квизов в аудитории или онлайн.
Работает прямо в браузере, не требует установки. Поддерживает режимы командной и одиночной игры, таймер, ставки, блиц-раунды, раунд «100 к 1» и живую таблицу лидеров.

---

## Архитектура

```
Игроки (hub.html) ──┐
                    ├── Supabase (PostgreSQL + Realtime) ──── Ведущий (presenter.html)
Проектор (projector.html) ──┘                                 Наблюдатель (observer.html)
```

Все данные хранятся в Supabase. Синхронизация через Realtime-подписки — ведущий пишет состояние, остальные экраны обновляются автоматически.

### Таблицы Supabase

| Таблица | Назначение |
|---------|-----------|
| `quiz_state` | Текущее состояние игры: фаза, индекс вопроса, показ ответа, флаги |
| `quiz_scores` | Очки игроков и команд |
| `quiz_chat` | Чат игроков, ответы на ставки и вопросы «100 к 1» |

## Игровые механики

### Стандартный раунд
- Таймер обратного отсчёта
- 4 варианта ответа
- Очки начисляются за правильный ответ с учётом скорости
- Серийный бонус: каждые 3 правильных ответа подряд — +0.5 очка

### Блиц-раунд
Быстрые вопросы без вариантов ответа — вводится свободный текст. Активируется ведущим между основными вопросами.

### Ставки (Bet-раунд)
Перед вопросом игрок делает ставку. При правильном ответе ставка удваивается, при неправильном — вычитается.

### Серийная кража очков
Если игрок даёт 5 правильных ответов подряд — у лидера таблицы автоматически похищается 1 очко, которое переходит к серийному игроку. Уведомление «СЕРИЯ 5! КРАЖА!» отображается на экране участника с анимацией.

### Раунд «100 к 1»
Аналог телевизионной игры «100 к 1» (Family Feud).

**Как проходит раунд:**
1. Ведущий нажимает «Начать 100к1» — игроки видят вопрос
2. Каждый игрок вводит свой вариант ответа (свободный текст) и выставляет ставку от 1 до 5 очков
3. Ведущий по очереди раскрывает популярные ответы кнопкой «Раскрыть →»
4. После раскрытия всех ответов ведущий нажимает «Завершить 100к1»
5. Очки начисляются:
   - Ответ совпал с одним из популярных → игрок получает очки по шкале популярности (5–4–3–2–1)
   - Ответ не совпал → из счёта вычитается размер ставки

**Вопросы «100 к 1» в квизах** (появляются в случайной позиции каждого квиза):

| Квиз | Школа | СПО |
|------|-------|-----|
| Экология | Способы защиты окружающей среды | Источники загрязнения атмосферы |
| История | Великие правители России | Причины революции 1917 года |
| Наука | Планеты Солнечной системы | Разделы физики |
| Патриотика | Русские народные богатыри | Достижения СССР |
| География | Реки России | Природные зоны России |
| Космос | Что видно в ночном небе | Типы небесных тел |
| Спорт | Командные виды спорта | Дисциплины лёгкой атлетики |
| Арктика | Животные Арктики | Природные ресурсы Арктики |

**Синхронизация:** Фаза раунда кодируется прямо в поле `phase` базы данных как `survey_0`, `survey_1`, … `survey_N`, где N — номер раскрытого ответа. Это позволяет синхронизировать состояние без изменения схемы БД.

## PWA

`presenter.html` установлен как Progressive Web App — ведущий может добавить его на главный экран телефона/планшета и использовать офлайн (вопросы закешированы Service Worker).

---

## Технологии

- **Frontend:** Vanilla JS, CSS Custom Properties, SVG-анимации
- **Backend:** [Supabase](https://supabase.com) (PostgreSQL + Realtime)
- **PWA:** Service Worker (`sw.js`, `sw-presenter.js`), Web App Manifest
- **Деплой:** Любой статический хостинг (GitHub Pages, Netlify и др.)

---

## Быстрый старт

1. Создать проект в Supabase и таблицы `quiz_state`, `quiz_scores`, `quiz_chat`
2. Прописать `SURL` и `SKEY` (anon key) в начале `hub.html`, `presenter.html`, `projector.html`
3. Разместить файлы на статическом хостинге
4. Ведущий открывает `presenter.html`, создаёт игру и сообщает код игры участникам
5. Игроки открывают `hub.html`, вводят код и имя/команду
6. На проекторе открывается `projector.html` с тем же кодом игры
